Origem do HTML

Voltar

O HTML (HyperText Markup Language) foi criado por Tim Berners-Lee em 1989 enquanto trabalhava no CERN (Conselho Europeu de Pesquisa Nuclear).

A intenção de Tim era criar uma maneira simples de compartilhar documentos científicos entre pesquisadores.

Ele desenvolveu o HTML como parte de um projeto maior que também incluía o HTTP (HyperText Transfer Protocol) e o URL (Uniform Resource Locator), formando a base da World Wide Web que conhecemos hoje.

A primeira versão do HTML era bastante simples e permitia criar páginas web estruturadas eficientemente. Com o tempo, o HTML evoluiu através de várias versões, sendo padronizado pelo World Wide Web Consortium (W3C) em 1995.

A versão mais recente, HTML5, lançada em 2014, trouxe muitas inovações, como suporte nativo para áudio e vídeo, elementos semânticos e capacidades de armazenamento local.

E assim, o HTML se tornou a pedra angular do desenvolvimento web, permitindo a criação de páginas e aplicações acessíveis globalmente.

Estrutura Básica do HTML

Voltar

  1. <!DOCTYPE html> Declara o tipo de documento e a versão do HTML (HTML5).

  2. <html lang="pt-br"> Inicia o documento HTML e define o idioma da página.

  3. <head> Contém metadados sobre o documento, como título, links para estilos e scripts.

  4. Exemplo de tags dentro de <head>:

    <meta charset="UTF-8">: Define a codificação de caracteres.

  5. <title> Define o título da página que aparece na aba do navegador.

  6. <body> Contém o conteúdo visível da página, como texto, imagens, links, listas, etc.

  7. Exemplo de tags dentro de<body> :

    <h1> a <h6>: Títulos e subtítulos.

    <p>: Parágrafos de texto.

    <a>: Links.

    <ul>: e <ol>: Listas não ordenadas e ordenadas..

    <img>: Imagens.

Oque é ClientSide

Voltar

Client-side se refere às operações realizadas no lado do cliente, ou seja, no navegador do usuário. Isso inclui a execução de scripts, a manipulação do Document Object Model (DOM) e a renderização de páginas web.

Tecnologias como HTML, CSS e JavaScript são usadas para criar e estilizar o conteúdo que o usuário vê e interage diretamente.

Em resumo, o client-side é responsável por:

  • Renderização de Interface: Exibição do layout e do design da página.
  • Interatividade: Respostas a cliques, teclas pressionadas e outros eventos do usuário.
  • Manipulação de Conteúdo: Adicionar, remover ou alterar elementos na página sem a necessidade de recarregar o navegador.

Essas operações tornam a experiência do usuário mais dinâmica e responsiva, pois muitas ações podem ser executadas sem a necessidade de comunicação com o servidor.

O que é ServerSide

Voltar

Server-side se refere às operações realizadas no servidor, antes de os dados serem enviados ao navegador do usuário. Essas operações incluem processamento de dados, execução de scripts e interação com bancos de dados.

Linguagens como PHP, Python, Ruby, Java e Node.js são comuns no desenvolvimento server-side.

Em resumo, o server-side é responsável por:

  • Processamento de Dados: Executar cálculos e lógica de negócios.
  • Interação com Bancos de Dados: Consultar, inserir, atualizar e excluir dados em um banco de dados.
  • Geração de Conteúdo Dinâmico: Criar páginas personalizadas com base nas requisições do usuário.
  • Segurança: Controlar acesso a dados e recursos, protegendo contra ameaças como SQL Injection e Cross-Site Scripting (XSS).

As operações server-side são essenciais para fornecer dados e lógica de negócios robusta e segura aos aplicativos web, garantindo que as informações sejam processadas de forma eficiente antes de serem entregues ao navegador do usuário.

Tipos de Servidores Web

Voltar

Servidor Web (Web Server): Hospeda e serve páginas web estáticas e dinâmicas, manipulando solicitações HTTP e HTTPS. Exemplos incluem Apache, Nginx, e Microsoft IIS. Este tipo de servidor é ideal para entregar conteúdo estático, como HTML, CSS, JavaScript, e imagens.

Servidor de Aplicações (Application Server): Executa aplicações web que necessitam de lógica de negócios complexa e manipulação de dados. Exemplos incluem Tomcat, JBoss, e WebLogic. Este servidor é utilizado para rodar aplicações dinâmicas que requerem processamento no servidor, como frameworks web (Java, Python, Ruby).

Servidor de Banco de Dados (Database Server): Armazena, recupera e gerencia dados utilizando sistemas de gerenciamento de banco de dados. Exemplos incluem MySQL, PostgreSQL, e MongoDB. É utilizado para suporte a aplicações que requerem armazenamento e consulta de dados dinâmicos e estruturados.

Servidor de Arquivos (File Server): Armazena e gerencia arquivos, facilitando o acesso e compartilhamento de dados. Exemplos incluem Samba e servidores FTP. Este tipo de servidor é utilizado para armazenamento e compartilhamento de arquivos grandes, documentos, e backups.
